    G.O.M.S.

    1. Introdução

    No início, propuseram um conjunto de modelos chamado de família GOMS para analisar o desempenho de usuários competentes de sistemas computacionais, realizando tarefas dentro da sua competência e sem cometer erros. Os modelos GOMS têm se mostrado úteis para prever o desempenho, ou seja, predizer o impacto de decisões de design no desempenho competente.

    2. O método

    O GOMS é um método para descrever uma tarefa e o conhecimento do usuário sobre como realizá-la em termos de objetivos ("Goals"), operadores ("Operators"), métodos ("Methods") e regras de seleção ("Selection Rules"). Sendo os objetivos as tarefas propriamente ditas, aquilo que o usuário deseja efetuar, os operadores as ações necessárias para atingir determinado objetivo, os métodos são um conjunto de operações, uma série de etapas, e por fim, as seleções são os diferentes modos que o usuário pode interagir com o sistema.

    3. Motivos da Escolha

    O GOMS foi adotado como nosso método de análise e decomposição de tarefas devido sua aplicabilidade em situações onde os usuários já tenham conhecimento dos métodos para a realização de tarefas, podendo ser usado tanto para prever e analisar a performance de usuários em suas realizações quanto para melhorar a eficácia em processos rotineiros.

    Tudo isso o faz uma ótima escolha para o projeto, pois estamos trabalhando sobre uma plataforma já construída, e com ele podemos, por meio de uma lista de objetivos de usuário, obtermos aqueles que podem estar utilizando métodos ineficientes de interação, inconsistências e outros problemas de design.

    4. Objetivos

    • 4.1 Resolver problemas de programação

      • GOAL 0: Resolver questão

        • GOAL 1: Acessar conta pessoal

        • METHOD 1.A: Acessar com uma conta já criada

          • GOAL 2: Efetuar Login

            • OP 2.1: Deslocar o mouse até o topo direito da página;

            • OP 2.2: Apertar com botão esquerdo do mouse em "Login";

            • OP 2.3: Preencher o campo de "Usuário";

            • OP 2.4: Preencher o campo de "Senha";

            • OP 2.5: Deslocar o mouse até o botão "Login";

            • OP 2.6: Apertar com o botão esquerdo do mouse.

        • METHOD 1.B: Acessar sem ter conta criada

          • GOAL 3: Se cadastrar

            • OP 3.1: Deslocar o mouse até o topo direito da página;

            • OP 3.2: Apertar com botão esquerdo do mouse em "Register";

            • OP 3.3: Preencher o campo de "Usuário";

            • OP 3.4: Preencher o campo de "Senha";

            • OP 3.5: Preencher o campo para confirmar "Senha";

            • OP 3.6: Preencher o campo de "Apelido";

            • OP 3.7: Preencher o campo de "Escola/Universidade";

            • OP 3.8: Preencher o campo de "QQ";

            • OP 3.9: Preencher o campo de "E-mail";

            • OP 3.10: Preencher o campo de "Captcha";

            • OP 3.11: Deslocar o mouse até o botão "Register";

            • OP 3.12: Apertar com o botão esquerdo do mouse.

          • GOAL 4: Visualizar questão

            • OP 4.1: Deslocar o mouse até o topo da página;

            • OP 4.2: Apertar com botão esquerdo do mouse em "Problems";

            • OP 4.3: Selecionar um juízo específico (ou todos) em "OJ";

            • OP 4.4: Deslocar o mouse até o título ou número do problema a ser visualizado;

            • OP 4.5: Apertar com botão esquerdo do mouse.

          • GOAL 5: Submeter resposta

            • OP 5.1: Deslocar o mouse até o topo esquerdo da página;

            • OP 5.2: Apertar com botão esquerdo do mouse em "SUBMIT";

            • OP 5.3: Deslocar o mouse até o botão "Select Language".

            • OP 5.4: Apertar com botão esquerdo do mouse na linguagem de programação quista;

            • OP 5.5: Apertar com botão esquerdo do mouse na opção "Yes" ou "No" em "Share".

            • OP 5.6: Colocar código programado em "Solution".
